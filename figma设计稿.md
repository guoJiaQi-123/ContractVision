# 企业销售合同数据可视化平台 - Figma规范化设计稿
## 设计总则
### 核心定位
面向企业销售/财务/管理层的合同数据可视化平台，基于Python技术栈（后端数据处理+Plotly/ECharts可视化渲染），核心价值为**合同数据实时监控、多维度分析、风险预警、权限管控**，设计风格遵循「企业级专业感+轻量化交互」，兼顾数据展示的直观性与操作的高效性。

### 设计原则
1. **专业性**：色彩/布局贴合企业办公场景，避免过度花哨，突出数据核心；
2. **易用性**：操作路径≤3步，高频功能一键触达，数据筛选/导出轻量化；
3. **新颖性**：采用卡片式模块化布局、动态图表交互、暗黑/亮色模式切换；
4. **兼容性**：优先适配桌面端（1920×1080为主），兼顾1366×768分辨率。

- 折线图：用于合同金额趋势、月度签约量；
- 饼图：用于合同类型占比、客户行业分布；
- 柱状图：用于部门/销售签约业绩对比；
- 漏斗图：用于合同从报价→签约→回款的转化；
- 预警仪表盘：用于合同逾期率、回款率的阈值监控。

- 响应式适配：侧边栏可折叠、表格列可自适应宽度、卡片布局自动换行。

## 四、技术衔接备注（Python+前端）
1. 可视化图表：Figma中图表为占位符，实际开发通过Python的Plotly/ECharts生成JSON数据，前端渲染；
2. 权限管控：用户管理页的权限配置需与Python后端的RBAC权限模型对接；
3. 数据导出：Python后端实现Excel/PDF生成，前端调用接口触发下载；
4. 预警通知：Python后端定时任务（Celery）检测阈值，对接邮件/短信API推送。
